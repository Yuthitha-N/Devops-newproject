CI/CD PIPELINE :

STEP 1: Push Your Project to GitHub

Your GitHub repo must contain:

Dockerfile
server.js
index.html
package.json

Push code:

git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/your-username/your-repo.git

git push -u origin main

STEP 2: Install Required Jenkins Plugins

Go to:

Manage Jenkins ‚Üí Manage Plugins

Install:

‚úî Git Plugin
‚úî Docker Pipeline Plugin

Restart Jenkins.

STEP 3: Create New Pipeline Job

Click New Item

Name: devops-pipeline

Select Pipeline

Click OK

STEP 4: Configure Pipeline

Scroll to Pipeline section.

Definition ‚Üí Select:

Pipeline script

Paste this pipeline code:

pipeline {
agent any

environment {
    IMAGE_NAME = "vaishh-node-app"
    CONTAINER_NAME = "vaishh-container"
}

stages {

    stage('Clone Repository') {
        steps {
            git 'https://github.com/your-username/your-repo.git'
        }
    }

    stage('Build Docker Image') {
        steps {
            sh 'docker build -t $IMAGE_NAME .'
        }
    }

    stage('Stop Old Container') {
        steps {
            sh 'docker rm -f $CONTAINER_NAME || true'
        }
    }

    stage('Run Docker Container') {
        steps {
            sh 'docker run -d -p 5000:5000 --name $CONTAINER_NAME $IMAGE_NAME'
        }
    }

}


}

Replace:

https://github.com/your-username/your-repo.git

with your real GitHub repository URL.

Click Save.

STEP 5: Run Pipeline

Click:

Build Now

Go to:

Console Output

You will see:

‚úî Cloning repo
‚úî Building Docker image
‚úî Running container

STEP 6: Access Your App

Open browser:

You will see your CI/CD webpage:

"Hello Vaishh ‚Äì Jenkins and Docker deployment running successfully"
